data-controller

data-controller require a controller name. This is the way how SixthJs supports Model-View-View-Model design pattern. This attribute specify the DOM element that will be bound to certain controller. Controller can be specified through attribute data-controller or in route service, when a route is registered. It is recommended to use one way or another, usage of both ways, in the DOM and in route service will initiate twice the controller.

MVVM components in SixthJs:

  • Model - Models are the properties specified with this in controller. This properties are specified in the DOM, and used through bindings.
  • View - The HTML that is rendered into the View
  • View - The HTML elements that changes the model (input, select, textarea)
  • Model - Properties attached and updated from the View

Usage:

data-controller is used as attribute:

          
            <div data-controller="MyController"></div>

            <script>
              sixth.controller('MyController', function(){
                this.user = 'Borg';
              });
            </script>
